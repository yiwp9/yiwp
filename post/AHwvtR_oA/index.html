成老师的带传感器信息的拍照想法

2022-01-10 4 min read

一、起源与功能介绍

来源

上周聊到无人机相关话题。成老师就提起他之前有写过一套算法，可以实现定位追踪，不过现在没法用了。后来把他四五年前的源码发给我，打开发现他的源码还是基于 Android5.0 的，各种 API已经改版没法使用了，于是就开启了我的改版之路。

功能

拍摄照片
  • 实时获取本机的传感器信息（陀螺仪、GPS等传感器）；
  • 定时拍照，同时保存拍照这一刻的传感器信息；
查看照片
  • 读取前面拍照时保存的图片的信息；
  • 监测当前位置与各传感器信息；
  • 对当前信息与照片信息进行对比，计算出需要做的动作（王某个方向移动多少步，传感器翻转多少角度）。
小总结

通过拍照与查看照片以及传感器信息对比，即可实现精确定位，通过户外实测，精度可达一米左右。

二、开始动手

准备工作

距离上次开发安卓已经有五六年的时间了，再次捡起来还是有点小激动。

  • 首先安装 Android Studio，这款软件是谷歌基于 IEDA 开发的，必属精品，用起来特别舒服；
  • 安装模拟器，因为成老师的手机是 Android11，于是我安装的模拟器也选择了 Android11。

开始干活

经分析发现存在如下问题：

  • Android 更新后授权方式改变，原本直接在 Manifest 文件申请权限就可以运行了，而在 Android6.0 之后采用运行是授权来放置权限滥用；
  • Camera 这个API 早在 API21 之时就已废除，因此在 Android11 版本上没法使用，会导致闪退；
  • 文件存储在新版 Android 也更新了，更加规范，更加易管理。因此不能随意存储，只能在指定的目录存储，或者在应用目录下进行存储；
  • 传感器信号获取相关代码也不能工作。

三、解决问题

针对发现的问题采用如下解决方案：

  • 更新授权方式为运行时授权，各种踩坑后决定采用 EasyPermissions 框架来管理权限；
  • 因为 Camera API 被废弃，便采用 CameraX，但是也出现了莫名其妙的问题（学艺不精），于是进一步偷懒，使用更更简单的 CameraView组件，几行代码便可实现照相机预览；
  • 文件存储则简单一点，只需要改改路径即可，参考 API 及很容易解决了。

五、总结

还存在的问题

1、首次启动时摄像头不显示，重新启动即可；
2、未拍照的情况想点击查看会闪退，后期修复；（已定位到原因为读取文件不存在）
3、闪光灯关闭打开未作处理；
4、停止拍摄后，再次点拍摄不起作用；
5、点击退出若界面为从上个界面不会直接退出app，而是返回上一界面，知道缓存中没有界面才能退出；（后期修改）
6、拍照间隔时间太短的话，通知会跟不上，造成无法停止拍照的错觉，其实拍照已经停止。（每拍一张照会弹一次拍照通知，一直按顺序弹，直到弹出的通知与照片数相对应）

使用的模块

1、EasyPermissions 权限管理模块
2、CameraView 相机组件，快速实现相机预览、拍照灯一系列操作（超级简单，三行代码即可实现预览）
